# Bank Flow Analyzer(银行流水分析专家)

小微信贷银行流水深度分析工具,帮助企业或信贷人员评估企业经营状况、识别风险点、分析资金流转模式。

## 执行流程

```
1. 数据解析与验证 → 识别流水格式 + 数据质量检查
2. 总体概况分析 → 计算关键指标 + 流入流出比例
3. 月度与季度分析 → 趋势识别 + 季节性分析
4. 风险识别与预警 → 异常检测 + 风险评分
5. 交易对手方分析 → 集中度分析 + 业务关联性 + 公司背景查询(企查查) + 合理性评估
6. 交叉验证 → 与财报对比 + 经营逻辑验证
```

## Step 1: 数据解析与验证

### 1.1 流水格式识别

| 数据来源 | 常见格式 | 解析要点 |
|---------|---------|---------|
| **银行流水单** | PDF/Excel/TXT | 日期、交易类型、金额、余额、摘要 |
| **网银导出** | Excel | 标准化字段映射 |
| **纸质流水扫描件** | 图片 | OCR识别 + 人工核对 |

### 1.2 数据质量检查

必须验证以下要素:
- ✅ 时间跨度:至少6个月,推荐12个月以上
- ✅ 账户类型:对公账户/个人账户/基本户/一般户
- ✅ 交易完整性:是否有断档、缺失月份
- ✅ 数据真实性:余额连续性、借贷平衡性

### 1.3 数据预处理

```
数据清洗步骤:
1. 去除重复交易记录
2. 标准化交易摘要(如"转账"、"提现"、"存款"等)
3. 识别并标记异常交易(如金额为0、负数等)
4. 按时间顺序排序
```

### 1.4 流水真实性验证(重要!)

根据《小微业务风控技术手册》,必须对银行流水进行真实性验证:

**验证方法一:银行官方验真**
- 检查流水是否盖有业务专用章
- 验证业务验证码或真伪验证二维码
- 登录银行官网或手机银行APP扫码验证

**验证方法二:支付宝银行卡转账验证**
- 登录支付宝-转账-转到银行卡
- 填写姓名和卡号进行核验
- 可验证流水所有人的真实性

**验证方法三:金额勾稽关系验证**
```
流水与征信报告匹配:
1. 银行流水还款信息 vs 征信报告贷款信息
2. 银行流水信用卡还款 vs 征信报告信用卡信息
3. 银行流水利息支出 vs 贷款本金匹配性

流水与经营资料匹配:
1. 银行流水房租支付 vs 租赁合同
2. 银行流水工资支付 vs 员工工资清单
3. 银行流水电费支付 vs 电费单
4. 银行流水应收应付 vs 交易合同、对账单
```

**验证方法四:假流水识别检查清单(新增!)**

根据实际案例分析,以下是识别假流水的关键检查项:

| 检查项 | 检查方法 | 风险等级 | 判定标准 | 假流水概率 |
|-------|---------|---------|---------|-----------|
| **余额计算验证** | 期末余额 - (期初余额 + 总收入 - 总支出) = 0 | ⭐⭐⭐⭐⭐ | 差异 > 0.01元 | 100% |
| **逻辑一致性检查** | 净流入符号 vs 余额变化方向 | ⭐⭐⭐⭐⭐ | 净流入<0且余额增加 | 100% |
| **利息合理性验证** | 日均余额 × 利率 vs 实际利息 | ⭐⭐⭐⭐⭐ | 日均>10万且利息=0 | 95% |
| **交易集中度检查** | 前五大收入对手占比 | ⭐⭐⭐⭐ | 单笔>总收入70% | 85% |
| **收入连续性检查** | 连续收入为0的月份数 | ⭐⭐⭐⭐ | 连续≥3个月 | 80% |
| **对手真实性验证** | 企查查查询主要交易对手 | ⭐⭐⭐⭐⭐ | 无法查询到 | 90% |
| **交易摘要分析** | 企业名称长度和规范性 | ⭐⭐⭐ | 名称>20字符且过于规范 | 70% |
| **交易时间合理性** | 周末节假日大额交易 | ⭐⭐⭐ | 非正常经营时间有大额交易 | 需核实 |
| **金额规律性检查** | 整数金额占比 | ⭐⭐⭐ | 整数金额占比>60% | 70% |
| **银行印章验证** | 检查银行公章 | ⭐⭐⭐⭐⭐ | 无公章或伪造公章 | 100% |

**假流水等级划分:**

| 等级 | 特征 | 识别难度 | 破绽数量 | 建议措施 |
|------|------|---------|---------|---------|
| **低级假流水** | 余额计算错误、逻辑矛盾明显、利息造假 | 低(基础数学验证) | ≥5个破绽 | 直接拒绝,标记可疑客户 |
| **中级假流水** | 余额正确、部分虚构、业务逻辑部分合理 | 中(需交叉验证) | 2-4个破绽 | 要求官方流水+实地调查 |
| **高级假流水** | 数据一致、逻辑合理、可能串通银行 | 高(需多方验证) | ≤1个破绽 | 银行官方验真+深度调查 |

**假流水识别流程:**
```
1. 快速检查: 余额计算 + 逻辑一致性 + 利息验证
   ↓ 发现破绽 → 判定为假流水 → 拒绝申请
   ↓
2. 深度检查: 交易集中度 + 收入连续性 + 对手真实性
   ↓ 发现破绽 → 要求官方流水 + 实地调查
   ↓
3. 综合判断: 结合行业特征 + 经营规模 + 访谈信息
   → 最终判定: 真实/可疑/虚假
```

**低级假流水典型特征(来自实际案例):**
```
✗ 期初余额35.88万,期末余额159.7万,但净流出128万(矛盾!)
✗ 日均余额174.76万,但季度利息为0元(不合理!)
✗ 74.18%的收入来自单笔交易133.9万(过于集中!)
✗ 连续4个月收入为0(业务中断不合理!)
✗ 交易摘要企业名称:"吴忠市绿洲牧场"、"利通区金银滩镇团庄村村民委员会"(过于规范!)
✗ 利息税记录间隔完全规律(每季度21日,金额0元)(人工痕迹明显!)
```

**经验教训:**
1. **余额计算是核心** - 这是识别假流水最有效的方法,任何偏差都是危险信号
2. **利息记录要验证** - 日均余额高但利息为0是明显的造假特征
3. **对手信息要核实** - 通过企查查查询交易对手真实性,无法查询的极可能是虚构
4. **业务逻辑要合理** - 正常经营企业不应该长时间收入为0
5. **集中度要警惕** - 单一对手占比过高风险极大

### 1.5 多账户合并分析

PDF特别强调需合并分析借款人及其关联人流水:

**需合并的账户范围:**
- ✅ 借款人本人银行账户
- ✅ 配偶银行账户
- ✅ 经营主体(企业)银行账户
- ✅ 参与经营的父母、成年子女账户
- ✅ 主要关联人账户

**合并分析原则:**
- 月日均存款:上述所有账户日均存款合计
- 如流水走财务、员工的,核实后也可计入
- 统一时间口径进行汇总分析

## Step 2: 总体概况分析

### 2.1 核心指标计算

| 指标 | 计算公式 | 正常范围(参考) |
|------|---------|----------------|
| **总收入金额** | Σ(所有收入交易金额) | - |
| **总支出金额** | Σ(所有支出交易金额) | - |
| **净流入** | 总收入 - 总支出 | ≥0 为佳 |
| **月均收入** | 总收入 / 总月份数 | 行业相关 |
| **月均支出** | 总支出 / 总月份数 | 行业相关 |
| **交易笔数** | 收入笔数 + 支出笔数 | - |
| **流入/流出比例** | 总收入 / 总支出 | ≥1 为健康 |

### 2.2 资金流动性分析

```
流入/流出比例解读:
- ≥1.2:资金充裕,现金流健康 ⭐⭐⭐⭐⭐
- 1.0-1.2:资金平衡,现金流良好 ⭐⭐⭐⭐
- 0.8-1.0:略显紧张,需关注 ⭐⭐⭐
- <0.8:流动性风险 ⚠️ 警告
- <0.6:严重流动性风险 ⛔ 高危
```

### 2.3 交易活跃度分析

```
活跃度评估:
- 高活跃:月均交易笔数 > 100笔
- 中活跃:月均交易笔数 50-100笔
- 低活跃:月均交易笔数 < 50笔

结合行业特征综合判断(如商贸行业活跃度应高于制造业)
```

### 2.4 日均存款与固定费用匹配分析(核心指标!)

**根据PDF第2585-2592行,这是现金流健康度的关键指标:**

**计算公式:**
```
日均存款 = Σ(每日余额) / 天数
月经营固定支出 = 月租金 + 月水电费 + 月工资 + 月金融机构利息 + 月社会集资利息等
日均存款与固定费用比 = 月日均存款 / 月经营固定支出
```

**判断标准:**
```
月日均存款/月经营固定支出:
- >1.5: 资金充裕,流动性优秀 ⭐⭐⭐⭐⭐
- 1.0-1.5: 资金平衡,流动性良好 ⭐⭐⭐⭐
- 0.5-1.0: 略显紧张,需关注 ⭐⭐⭐
- <0.5: 流动性紧张,高风险 ⚠️ 警告
```

**除夕余额分析(特殊关注):**
```
除夕余额比 = 除夕余额 / 月日均存款

- >1.5: 年终资金储备能力强 ⭐⭐⭐⭐⭐
- 1.0-1.5: 年终储备正常 ⭐⭐⭐⭐
- <1.0: 年终储备不足 ⚠️

分析要点:
1. 结合应收应付情况分析除夕余额合理性
2. 关注除夕前后时间段的大额资金流动
3. 分析集中支付后是否仍有盈余
```

### 2.5 销售回款比分析

**根据PDF第1236行,这是评估经营质量的关键指标:**

**计算公式:**
```
销售回款比 = 实际现金回款 / 营业收入 × 100%
实际现金回款 = 银行流水统计的销售回款金额
营业收入 = 客户申报的年度/季度销售收入
```

**判断标准:**
```
销售回款比:
- ≥90%: 回款良好,现金流健康 ⭐⭐⭐⭐⭐
- 80%-90%: 基本正常,应收适度 ⭐⭐⭐⭐
- 70%-80%: 应收增加,需关注 ⚠️
- <70%: 回款风险,流动性压力 ⛔ 高危

分析要点:
1. 对应资产负债表中应收账款余额变化
2. 分析应收账款回收周期
3. 关注是否存在大量赊销情况
4. 结合行业平均回款周期对比
```

### 2.6 账户行为特征分析

**识别账户使用模式,发现潜在风险信号:**

**存款模式分析:**
```
- 定期存款:每月固定日期收入(如工资日、结算日)
- 不定期存款:收入时间不规律
- 大额存款:单笔存款超过月均收入3倍
- 小额存款:频繁的小额收入

风险提示:
- 只有提现无存款,可能用于民间借贷利息支付
- 大额整数存款,可能为借款或其他资金流入
```

**转账规律分析:**
```
- 固定日期转账:每月固定时间转出固定金额(可能为还款)
- 账户间频繁往来:多个账户间频繁资金流动(可能为过账)
- 特殊摘要:备注"借款"、"利息"、"还款"等

民间借贷识别特征:
⚠️ 每月/每季给固定账号支付固定金额
⚠️ 交易备注出现"借款"、"利息"、"还款"等关键词
⚠️ 大额整数频繁转出转入(非正常经营交易)
⚠️ 对象非供应商或客户,且无合理业务关系
```

## Step 3: 月度与季度分析

### 3.1 月度收支明细

生成月度收支表格,包括:
```
月份 | 收入金额 | 支出金额 | 交易笔数 | 净流入 | 余额变化
```

### 3.2 季节性分析

按季度汇总,识别季节性特征:

```markdown
### 季度收支汇总

**Q1(1-3月)**
- 收入:X元
- 支出:Y元
- 净流入:Z元
- 特征:淡季/旺季/平稳

**Q2(4-6月)**
- ...
```

### 3.3 收入波动性分析

```
变异系数计算:
CV = 标准差 / 均值

稳定性判断:
- CV < 0.3:收入稳定 ⭐⭐⭐⭐⭐
- CV 0.3-0.5:基本稳定 ⭐⭐⭐⭐
- CV 0.5-0.7:波动较大 ⭐⭐⭐
- CV > 0.7:波动剧烈 ⚠️ 风险提示
```

### 3.4 趋势分析

识别收入/支出的长期趋势:
- **上升趋势**:连续3个月以上增长
- **下降趋势**:连续3个月以上下降
- **波动趋势**:无明显规律
- **季节性波动**:特定月份明显偏离均值

## Step 4: 风险识别与预警

### 4.0 假流水识别与风险等级(新增!)

根据实际案例分析,假流水识别是风控工作的重中之重。以下是系统化的假流水识别框架:

**假流水识别的"三大必查":**

1. **余额计算验证(必查,零容忍)**
   ```
   公式: 期末余额 = 期初余额 + 总收入 - 总支出
   判定: 差异 > 0.01元 → 100%假流水
   ```
   案例: 期初35.88万 + 收入180.5万 - 支出308.58万 = -92.2万
         实际显示: 159.7万 (相差251.9万!) → 假流水

2. **逻辑一致性检查(必查,零容忍)**
   ```
   公式: 净流入 = 总收入 - 总支出
   判定: 净流入<0 且 期末余额>期初余额 → 100%假流水
   ```
   案例: 净流出128万,但余额反而增加123万 → 逻辑矛盾 → 假流水

3. **利息合理性验证(必查,零容忍)**
   ```
   公式: 预期季度利息 = 日均余额 × 0.003(假设0.3%季度利率)
   判定: 日均>10万且利息=0 → 95%假流水
   ```
   案例: 日均174.76万,但季度利息为0元 → 假流水

**假流水识别的"七大预警信号":**

| 信号 | 描述 | 破绽度 | 判定标准 | 假流水概率 |
|------|------|--------|---------|-----------|
| 🔴 **余额算不平** | 期末余额 ≠ 期初+收入-支出 | 致命 | 差异>0.01元 | 100% |
| 🔴 **逻辑矛盾** | 净流出但余额增加 | 致命 | 净流入<0且余额增加 | 100% |
| 🔴 **利息为0** | 日均高但无利息 | 致命 | 日均>10万且利息=0 | 95% |
| 🟠 **交易过于集中** | 单一对手占比过高 | 高危 | 单笔>总收入70% | 85% |
| 🟠 **收入中断** | 连续多月无收入 | 高危 | 连续≥3个月 | 80% |
| 🟠 **对手虚构** | 企查查查不到对手 | 高危 | 查询无结果 | 90% |
| 🟡 **摘要过于规范** | 企业名称过于完整 | 中危 | 名称>20字符且规范 | 70% |

**假流水识别的"十大检查清单":**

```
✅ 检查1: 余额计算
   计算: 期末余额 - (期初余额 + 总收入 - 总支出)
   判定: ≠0 → 假流水

✅ 检查2: 逻辑一致性
   计算: 净流入 = 总收入 - 总支出
   判定: 净流入<0且期末余额>期初余额 → 假流水

✅ 检查3: 利息验证
   计算: 预期利息 = 日均余额 × 0.003
   判定: 预期>0但实际利息=0 → 假流水

✅ 检查4: 交易集中度
   计算: 前五大收入对手占比
   判定: 单一对手>70% → 需深度核实

✅ 检查5: 收入连续性
   统计: 连续收入为0的月份数
   判定: 连续≥3个月 → 需核实业务合理性

✅ 检查6: 对手真实性
   操作: 企查查查询前五大对手
   判定: 无法查询 → 假流水风险极高

✅ 检查7: 摘要规范性
   观察: 企业名称长度和完整性
   判定: 过于规范完整 → 需警惕

✅ 检查8: 交易时间
   检查: 周末节假日大额交易
   判定: 非经营时间有大额交易 → 需核实

✅ 检查9: 金额规律
   统计: 整数金额占比
   判定: >60% → 真实性存疑

✅ 检查10: 银行印章
   检查: 是否有银行公章
   判定: 无公章或伪造 → 假流水
```

**假流水分级与处理:**

| 等级 | 特征 | 识别难度 | 处理措施 |
|------|------|---------|---------|
| **低级假流水** | 余额算不平、逻辑矛盾、利息造假 | ⭐ 简单(基础数学) | ❌ 直接拒绝,标记客户 |
| **中级假流水** | 余额正确、部分虚构、业务部分合理 | ⭐⭐ 中等(需交叉验证) | ⚠️ 要求官方流水+实地调查 |
| **高级假流水** | 数据一致、逻辑合理、可能串通银行 | ⭐⭐⭐ 困难(需多方验证) | 🔍 银行官方验真+深度调查 |

**实际案例经验(2026-02-04):**

案例: 马建林银行流水深度分析

**核心破绽:**
1. 期初35.88万 + 收入180.5万 - 支出308.58万 = -92.2万,但期末显示159.7万
2. 净流出128万,但余额反而增加123万
3. 日均余额174.76万,但季度利息为0元
4. 74.18%收入来自单笔133.9万(青储款)
5. 连续4个月(2025年5-8月)收入为0
6. 企业名称:"吴忠市绿洲牧场"、"利通区金银滩镇团庄村村民委员会"(过于规范)
7. 利息税记录:每季度21日,间隔完全规律,金额0元(明显造假)

**结论:** 低级假流水,破绽明显,经不起推敲

**经验总结:**
- 假流水制作者往往直接编写期初期末余额,未验证数学逻辑
- 会添加一些"看似真实"的电子支付记录,但核心数据矛盾
- 企业名称设计过于完整规范,不符合日常流水特征
- 利息记录造假明显,对真实利息计算不了解

**应对措施:**
1. 凡是余额算不平的,直接判定为假流水
2. 凡是逻辑矛盾的,直接判定为假流水
3. 凡是利息不合理的,直接判定为假流水
4. 发现假流水,立即拒绝贷款申请
5. 标记可疑客户,防止再次申请

### 4.3 隐性负债识别(重要!)

**根据PDF第1157-1160行,通过银行流水识别隐性负债:**

**非银行借款识别信号:**
```
高危险信号 ⚠️:
1. 摘要中出现"小额贷款"、"利息"、"还款"等关键词
2. 每月/每季给固定账号支付固定金额
3. 大额整数频繁转出转入
4. 流水余额波动剧烈,无正常经营收入支撑
5. 只有提现无存款,可能用于支付利息
6. 交易对手为非供应商/客户,且无业务关联

分析要点:
1. 结合后续有无定期支付利息判断
2. 核对征信报告,确认是否为未上征信贷款
3. 分析交易对手背景,判断是否为借贷关系
4. 计算隐性负债规模和利息负担
```

**隐性负债影响评估:**
```
隐性负债判断标准:
- 月均隐性利息支出 > 月均净利润的30%: 高风险 ⛔
- 月均隐性利息支出 > 月均净利润的20%: 中高风险 ⚠️
- 月均隐性利息支出 > 月均净利润的10%: 需关注

综合影响:
1. 隐性负债会加剧资金周转压力
2. 利息负担会侵蚀经营利润
3. 影响再融资能力和信用评级
```

### 4.5 上下游结算规律分析

**根据PDF第3316行,分析账款结算情况:**

**结算模式识别:**
```
上游供应商结算:
- 现结为主:付款后立即收款,现金流压力小
- 账期结算:存在一定账期,需关注应付账款
- 预付模式:提前付款,资金占用时间长
- 混合结算:不同供应商不同模式

下游客户结算:
- 预收为主:客户预付款,现金流充裕
- 现结为主:款到发货,现金流健康
- 赊销为主:存在应收账款,关注回款
- 混合结算:不同客户不同模式
```

**结算质量评估:**
```
结算质量指标:
1. 上游账期平均天数 vs 行业平均水平
2. 下游回款平均天数 vs 行业平均水平
3. 应收账款回款率(实际回款/应收余额)
4. 应付账款付款率(实际付款/应付余额)

风险信号 ⚠️:
1. 上游账期明显短于行业(资金压力)
2. 下游回款明显慢于行业(现金流紧张)
3. 应收账款持续增长且回款慢
4. 大额逾期应收账款
5. 频繁催收应付账款
```

### 企查查信息真实性验证

**交叉验证要点**:
```markdown
1. 公司名称一致性:
   - 流水中的公司名称 vs 企查查公司名称
   - 注意简称vs全称、繁简体等

2. 经营状态验证:
   - 企查查显示"存续" ✓
   - 企查查显示"吊销/注销" ⚠️ 风险

3. 时间一致性:
   - 公司成立时间 vs 交易开始时间
   - 交易时间应在公司成立之后

4. 经营范围匹配:
   - 交易内容 vs 公司经营范围
   - 应在经营范围内或相关联

5. 交易金额合理性:
   - 交易金额 vs 注册资本/公司规模
   - 明显超出规模需核实
```

### 无法查询到的处理

**情况1: 公司名称输入错误**
```
处理方法:
- 尝试模糊搜索
- 结合经营范围、注册地等信息匹配
- 要求客户提供公司全称
```

**情况2: 公司已注销很久**
```
处理方法:
⚠️ 高风险!
- 注销后不应有正常交易
- 可能是虚构交易或历史流水
- 必须核实交易真实性
```

**情况3: 公司根本不存在**
```
处理方法:
⚠️ 极高风险!
- 可能是虚构交易对手
- 必须提供合同、发票等凭证
- 如无法提供,建议拒贷
```
