# 个人背景调查技能 - 快速开始指南

## 简介

本技能用于查询个人的司法信息，包括：
- **中国执行信息公开网** - 查询被执行人、失信记录、限制消费令
- **中国裁判文书网** - 查询民事、刑事、行政案件

## 快速使用

### 1. 基础查询

```bash
cd /Users/jj/CodeBuddy/skill/personal-background-check

python scripts/simple_dual_query.py --name "张三" --id-card "110101199001011234"
```

### 2. 参数说明

| 参数 | 简写 | 必填 | 说明 |
|------|------|------|------|
| `--name` | `-n` | 是 | 查询姓名 |
| `--id-card` | `-i` | 是 | 查询身份证号（18位） |
| `--headless` | - | 否 | 无头模式（不显示浏览器窗口） |
| `--max-attempts` | `-m` | 否 | 最大尝试次数（默认10次） |

### 3. 使用示例

#### 示例1：基本查询
```bash
python scripts/simple_dual_query.py -n "张三" -i "110101199001011234"
```

#### 示例2：无头模式（后台运行）
```bash
python scripts/simple_dual_query.py -n "张三" -i "110101199001011234" --headless
```

#### 示例3：增加尝试次数
```bash
python scripts/simple_dual_query.py -n "张三" -i "110101199001011234" -m 15
```

## 工作原理

### 自动重试机制

查询失败时自动重试：
1. **检测反爬虫** - 自动识别验证码、频率限制等情况
2. **指数退避** - 等待时间逐渐增加（2秒→4秒→8秒...）
3. **重新初始化** - 每次重试前重新启动浏览器
4. **最大尝试** - 默认最多尝试10次

### 人类行为模拟

- 随机打字速度（模拟真人输入）
- 随机页面滚动（模拟浏览行为）
- 随机等待时间（避免规律操作）
- 真实用户代理（模拟正常浏览器）

## 常见问题

### Q1: 查询失败怎么办？

A: 脚本会自动重试直到成功。如果超过最大尝试次数：
1. 检查网络连接
2. 查看 `screenshots/` 目录中的截图了解失败原因
3. 稍后重试（避免短时间内多次查询）

### Q2: 如何查看详细结果？

A: 
1. 查看控制台输出的摘要信息
2. 打开 `reports/` 目录下的 JSON 文件
3. 查看 `screenshots/` 目录下的截图

### Q3: 无头模式有什么用？

A: 
- **默认模式** (`--headless=False`)：显示浏览器窗口，可观察查询过程
- **无头模式** (`--headless`)：后台运行，适合自动化脚本

### Q4: 为什么需要自动重试？

A: 因为目标网站有反爬虫机制：
- 访问频率限制
- 验证码验证
- IP限制

自动重试可以：
- 增加等待时间规避频率限制
- 重新初始化浏览器规避检测
- 持续尝试直到成功
