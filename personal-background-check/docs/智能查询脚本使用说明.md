# 智能司法查询脚本使用说明

## 概述

`intelligent_judicial_query.py` 是一个优化的司法信息查询脚本，能够自动查询直到获得结果。

## 主要特性

### 1. 多策略自动回退
- **策略1**: undetected-chromedriver（专业反检测库）
- **策略2**: 标准 Selenium + CDP 命令（隐藏 webdriver 特征）

### 2. 人类行为模拟
- 随机打字延迟（0.05-0.2秒/字符）
- 随机滚动行为
- 随机点击延迟（0.5-1.5秒）
- 随机等待时间（2-10秒）

### 3. 智能重试机制
- 每个策略最多重试 3 次
- 遇到反爬虫页面自动等待后重试
- 失败后自动切换到下一个策略

### 4. 反检测技术
- 使用 undetected-chromedriver（如已安装）
- CDP 命令注入隐藏 `navigator.webdriver`
- 禁用自动化控制标志
- 真实用户代理和浏览器特征

### 5. 完善的日志
- 截图保存失败场景
- 详细的操作步骤记录
- 查询结果汇总报告

## 安装依赖

```bash
pip install -r requirements.txt
```

关键依赖：
- `selenium>=4.8.0` - Web 自动化
- `undetected-chromedriver>=3.5.0` - 反检测驱动（可选）

## 使用方法

### 基本使用

```bash
python scripts/intelligent_judicial_query.py
```

### 修改查询信息

编辑脚本中的 `main()` 函数：

```python
def main():
    name = "目标姓名"      # 修改为目标姓名
    id_card = "目标身份证号"  # 修改为目标身份证号（18位）
    
    query = IntelligentJudicialQuery(headless=False, max_retries=3)
    results = query.query_all(name, id_card)
```

### 参数说明

#### IntelligentJudicialQuery 参数

- `headless` (bool, 默认: False)
  - `True`: 无头模式，不显示浏览器窗口
  - `False`: 显示浏览器操作过程（推荐用于调试）

- `max_retries` (int, 默认: 3)
  - 每个策略的最大重试次数
  - 建议范围: 2-5

## 查询类型

脚本会依次查询以下三类信息：

1. **被执行人信息** - `https://zxgk.court.gov.cn/zhixing/`
2. **失信被执行人名单** - `https://zxgk.court.gov.cn/shixin/`
3. **限制消费令** - `https://zxgk.court.gov.cn/xgl/`

## 工作流程

```
开始查询
    ↓
初始化 undetected-chromedriver（策略1）
    ↓
访问查询页面
    ↓
检测反爬虫页面？
    ├─ 是 → 等待5-10秒 → 重试
    └─ 否 → 继续
        ↓
    模拟人类输入（姓名、身份证号）
    ↓
    点击查询按钮
    ↓
    等待结果加载
    ↓
    分析查询结果
    ├─ 成功 → 保存结果
    └─ 失败/超时 → 重试
        ↓
    达到最大重试次数？
        ├─ 是 → 尝试策略2（标准 Selenium）
        └─ 否 → 继续当前策略重试
            ↓
        策略2也失败？
            ├─ 是 → 记录失败，结束
            └─ 否 → 成功，保存结果
```

## 常见问题

### 1. undetected-chromedriver 安装失败

**问题**: `ImportError: No module named 'undetected_chromedriver'`

**解决**:
```bash
pip install undetected-chromedriver
```

### 2. Chrome 驱动版本不匹配

**问题**: `This version of ChromeDriver only supports Chrome version xxx`

**解决**:
- 更新 Chrome 浏览器到最新版本
- 或让 undetected-chromedriver 自动下载匹配版本

### 3. 反爬虫检测频繁

**问题**: 频繁遇到验证码或反爬虫页面

**建议**:
- 增加重试间隔时间
- 减少 `max_retries` 次数
- 分时段查询，避免短时间内多次查询

### 4. 查询超时

**问题**: 页面加载超时或查询无响应

**解决**:
- 检查网络连接
- 增加页面加载超时时间
- 使用非 headless 模式观察浏览器行为

## 注意事项

1. **法律合规**
   - 仅限用于合法的个人背景调查
   - 不得用于非法用途
   - 遵守相关法律法规

2. **使用频率**
   - 避免短时间内大量查询
   - 建议每次查询间隔至少 10 分钟
   - 频繁查询可能触发反爬机制

3. **数据准确性**
   - 自动化查询结果仅供参考
   - 重要信息应以官方最终记录为准
   - 建议结合手动查询进行验证

4. **隐私保护**
   - 查询结果文件包含敏感信息
   - 妥善保管结果文件
   - 避免泄露他人信息

## 与其他脚本的区别

## 技术细节

### CDP 命令注入

使用 Chrome DevTools Protocol (CDP) 注入 JavaScript，隐藏自动化特征：

```javascript
Object.defineProperty(navigator, 'webdriver', {
    get: () => undefined
});

Object.defineProperty(navigator, 'plugins', {
    get: () => [1, 2, 3, 4, 5]
});

window.chrome = { runtime: {} };
```

### 人类行为模拟

- **打字**: 每个字符间隔 0.05-0.2 秒
- **滚动**: 每次滚动 100-500 像素，间隔 0.3-0.8 秒
- **点击**: 点击前延迟 0.5-1.5 秒
- **等待**: 随机等待 2-10 秒
