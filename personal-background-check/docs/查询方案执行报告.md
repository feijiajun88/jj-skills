#### 1. 网站反爬虫机制
网站使用多层反爬虫保护：

**第一层：HTTP 412 响应**
- 使用 requests 库直接访问时，返回 HTTP 412 状态码
- 响应内容为简单的 HTML，包含大量混淆的 JavaScript 代码
- JavaScript 文件路径：`/U52nf4AkCaDm/93W4UXnYb8SB.bbf9512.js`

**第二层：JavaScript 验证**
- 页面需要执行 JavaScript 才能渲染真实内容
- JavaScript 代码用于验证客户端环境
- 验证失败则页面内容为空

**第三层：自动化检测**
- 检测 `navigator.webdriver` 属性
- 检测浏览器插件特征
- 检测自动化工具特征

#### 2. Selenium 遇到的问题

**测试日志显示：**
```

输入框数量: 0
Body内容长度: 0
```

**问题表现：**
- 页面状态显示加载完成（complete）
- 但 body 内容为空
- 无法找到任何输入框
- JavaScript 被执行但内容未渲染

**技术分析：**
网站的反爬虫 JavaScript 可能检测到：
1. ChromeDriver 的特征
2. 浏览器环境的异常
3. 时间和行为模式的异常
4. 某些浏览器 API 的缺失
