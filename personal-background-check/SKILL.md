# 司法信息查询技能 (Judicial Information Query)

## 技能概述

本技能通过中国执行信息公开网和裁判文书网查询个人司法信息。输入姓名和身份证号后，自动查询这两个网站直到获取结果，并保存查询截图。

适用于客户背景调查（KYC/尽职调查）、个人风险筛查、供应商/合作伙伴背景核查等场景。

---

## 核心功能

### 双站点司法信息查询

| 平台 | 查询内容 | 网址 |
|------|----------|------|
| 中国执行信息公开网 | 被执行人信息、失信记录、限制消费令 | https://zxgk.court.gov.cn/zhzxgk/ |
| 中国裁判文书网 | 民事、刑事、行政案件裁判文书 | https://wenshu.court.gov.cn |

### 特性

- ✅ **自动重试**：查询失败自动重试，直到成功或达到最大尝试次数
- ✅ **截图保存**：自动保存查询过程和结果截图
- ✅ **反爬虫策略**：智能检测并规避反爬虫机制
- ✅ **命令行参数**：支持通过命令行输入姓名和身份证号

## 使用方法

### 快速开始

```bash
# 使用简化双站点查询脚本（推荐）
python scripts/simple_dual_query.py --name "张三" --id-card "110101199001011234"

# 无头模式（后台运行，不显示浏览器窗口）
python scripts/simple_dual_query.py --name "张三" --id-card "110101199001011234" --headless

# 设置最大尝试次数（默认10次）
python scripts/simple_dual_query.py --name "张三" --id-card "110101199001011234" --max-attempts 15
```

### 参数说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `--name` | `-n` | 查询姓名（必填） | - |
| `--id-card` | `-i` | 查询身份证号（必填，18位） | - |
| `--headless` | - | 无头模式（不显示浏览器窗口） | False |
| `--max-attempts` | `-m` | 最大尝试次数 | 10 |

## 查询机制

### 自动重试策略

查询失败时自动重试，采用指数退避算法：

| 尝试次数 | 等待时间 | 说明 |
|----------|----------|------|
| 第1次 | 立即执行 | 发起查询 |
| 第2次 | 2-7秒 | 首次重试 |
| 第3次 | 4-9秒 | 二次重试 |
| 第4次 | 8-13秒 | 三次重试 |
| ... | 递增 | 持续重试 |
| 第10次 | 约60秒 | 最后尝试 |

**特点**：
- 检测到反爬虫时自动增加等待时间
- 每次重试前重新初始化浏览器驱动
- 可配置最大尝试次数（默认10次）

### 人类行为模拟

- 随机打字延迟（0.05-0.15秒/字符）
- 随机页面滚动（模拟真人浏览）
- 随机等待时间（1-5秒）
- 真实用户代理轮换

### 反爬虫检测

自动检测以下情况并处理：
- 验证码页面
- 访问频率限制
- 安全验证页面
- IP封禁提示
