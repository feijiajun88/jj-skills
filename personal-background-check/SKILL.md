# 司法信息查询技能 (Judicial Information Query)

## 技能概述

本技能通过整合中国各级法院的公开信息平台，查询个人司法信息，包括被执行人信息、失信被执行人名单、限制消费令和裁判文书等信息。

适用于客户背景调查（KYC/尽职调查）、个人风险筛查、供应商/合作伙伴背景核查等场景。

---

## 核心功能

### 司法信息查询（法院系统）

| 平台 | 查询内容 | 优先级 |
|------|----------|--------|
| 中国执行信息公开网 | 被执行人信息、限制消费令 | 🔴 核心 |
| 失信被执行人名单 | 失信行为记录 | 🔴 核心 |
| 中国裁判文书网 | 民事、刑事、行政案件 | 🔴 核心 |

## 使用方法

### 基本用法

```bash
# 使用智能查询脚本（推荐）
python scripts/intelligent_judicial_query.py

# 测试网站访问
python scripts/test_website_access.py

# 使用增强版查询脚本
python scripts/enhanced_judicial_query.py
```

### 参数说明

智能查询脚本参数：
- 在脚本中直接修改 `name` 和 `id_card` 变量
- `headless`: 是否无头模式（False 可看到浏览器操作）
- `max_retries`: 每个策略的最大重试次数

## 查询机制

### 重试策略

每个查询平台采用渐进式重试机制：

| 重试次数 | 间隔时间 | 动作 |
|----------|----------|------|
| 第1次 | 立即执行 | 发起查询 |
| 第2次 | 1秒后 | 首次重试 |
| 第3次 | 2秒后 | 二次重试 |
| 第4次 | 5秒后 | 三次重试 |
| 第5次 | 10秒后 | 四次重试 |
| 第6次 | 30秒后 | 最后重试 |

若6次均失败，记录失败原因并继续其他查询。

### 并行查询

- 不同平台间并行查询，提高效率
- 同一平台的多个子查询串行执行，避免触发反爬机制

---

## 查询机制

### 多策略自动回退

1. **策略1**: undetected-chromedriver（专业反检测库）
2. **策略2**: 标准 Selenium + CDP 命令（隐藏 webdriver 特征）

### 人类行为模拟

- 随机打字延迟（0.05-0.2秒/字符）
- 随机滚动行为
- 随机点击延迟（0.5-1.5秒）
- 随机等待时间（2-10秒）

### 智能重试机制

- 每个策略最多重试 3 次
- 遇到反爬虫页面自动等待后重试
- 失败后自动切换到下一个策略

## 数据来源说明

本技能所有数据均来源于官方公开平台：

| 平台 | 官网 | 数据类型 |
|------|------|----------|
| 中国执行信息公开网 | zxgk.court.gov.cn | 被执行人信息 |
| 中国裁判文书网 | wenshu.court.gov.cn | 司法文书 |

**重要提示**：裁判文书网有严格的反爬机制，建议使用 Selenium 自动化查询或第三方API服务。
