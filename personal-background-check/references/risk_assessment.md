# 风险评估标准与规则

本文档详细说明个人背景调查中的风险评估模型、评分规则和评级标准。

---

## 一、风险等级定义

### 风险等级体系

| 等级 | 标识 | 分数范围 | 定义 | 处理建议 |
|------|------|----------|------|----------|
| **无风险** | 🟢 | 90-100分 | 无任何负面记录 | 正常合作/录用 |
| **低风险** | 🟡 | 70-89分 | 存在轻微负面记录 | 常规关注 |
| **中风险** | 🟠 | 40-69分 | 存在中等负面记录 | 重点关注，谨慎决策 |
| **高风险** | 🔴 | 0-39分 | 存在严重负面记录 | 拒绝合作/录用 |

### 风险等级说明

#### 🟢 无风险 (90-100分)
- **特征**：无任何负面记录
- **表现**：征信良好，无司法纠纷，无行政处罚
- **建议**：可以正常合作或录用

#### 🟡 低风险 (70-89分)
- **特征**：存在1-2项轻微负面记录
- **表现**：
  - 有1-2起民事纠纷（非败诉方）
  - 轻微交通违法（1-2次）
  - 其他轻微不良记录
- **建议**：常规关注，不影响主要决策

#### 🟠 中风险 (40-69分)
- **特征**：存在多项负面记录或1-2项较严重记录
- **表现**：
  - 作为被告参与多起民事案件
  - 有行政处罚记录
  - 存在税务异常
  - 有3次以上交通违法
- **建议**：需要重点关注，深入了解具体情况后再做决策

#### 🔴 高风险 (0-39分)
- **特征**：存在严重负面记录
- **表现**：
  - 失信被执行人
  - 被执行人（有执行标的）
  - 限制高消费
  - 涉及刑事案件
  - 严重税务违法
- **建议**：建议拒绝合作或录用

## 二、风险评分模型

### 基础评分

```
初始分数：100分

评分公式：
最终分数 = 初始分数 - ∑(各项扣分)

最低分数：0分（不倒扣）
```

### 扣分细则

#### 1. 司法风险扣分

| 风险类型 | 扣分标准 | 说明 |
|----------|----------|------|
| 被执行人记录 | -30分/条 | 有执行案件未结案 |
| 失信被执行人 | -50分/条 | 被纳入失信名单 |
| 限制消费令 | -40分/条 | 被限制高消费 |
| 刑事案件（被告人） | -50分/条 | 涉及刑事犯罪 |
| 刑事案件（其他当事人） | -30分/条 | 与刑事案件有关联 |
| 民事案件（败诉方） | -20分/条 | 作为败诉方 |
| 民事案件（胜诉方） | -5分/条 | 作为胜诉方（有纠纷记录） |
| 民事案件（其他当事人） | -10分/条 | 作为原告/第三人 |
| 行政案件（败诉） | -25分/条 | 行政诉讼败诉 |
| 行政案件（其他） | -10分/条 | 其他行政诉讼情况 |

**封顶扣分**：司法类最高扣80分

#### 2. 信用风险扣分

| 风险类型 | 扣分标准 | 说明 |
|----------|----------|------|
| 征信不良记录（严重） | -40分 | 连续逾期90天以上 |
| 征信不良记录（一般） | -25分 | 逾期30-90天 |
| 征信查询频繁 | -10分 | 近3个月查询超5次 |
| 大额贷款逾期 | -35分 | 逾期金额较大 |
| 信用卡恶意透支 | -30分 | 恶意透支记录 |
| 担保代偿 | -20分 | 为他人担保代偿 |
| 欠税记录 | -30分/条 | 存在欠税行为 |
| 税务违法 | -40分/条 | 重大税务违法 |

**封顶扣分**：信用类最高扣60分

#### 3. 行政风险扣分

| 风险类型 | 扣分标准 | 说明 |
|----------|----------|------|
| 行政处罚（严重） | -30分/条 | 罚款5万元以上 |
| 行政处罚（一般） | -15分/条 | 罚款1-5万元 |
| 行政处罚（轻微） | -5分/条 | 罚款1万元以下 |
| 经营异常名录 | -20分/条 | 企业/个体户列入异常名录 |
| 严重违法失信 | -40分/条 | 列入严重违法失信名单 |
| 吊销许可证 | -35分/条 | 被吊销经营许可 |

**封顶扣分**：行政类最高扣50分

#### 4. 其他风险扣分

| 风险类型 | 扣分标准 | 说明 |
|----------|----------|------|
| 严重交通违法 | -15分/条 | 酒驾、肇事逃逸等 |
| 一般交通违法 | -5分/条 | 超速、闯红灯等 |
| 驾驶证异常 | -10分 | 吊销、暂扣等 |
| 社保异常 | -15分 | 长期断缴、欠费 |
| 多个企业异常 | -10分/个 | 关联企业异常 |

**封顶扣分**：其他类最高扣30分

### 加分项（风险缓解因素）

| 加分类型 | 加分标准 | 说明 |
|----------|----------|------|
| 被执行人已结案 | +15分/条 | 执行案件已履行完毕 |
| 失信已撤销 | +20分/条 | 已从失信名单中移除 |
| 案件胜诉 | +5分/条 | 作为原告胜诉 |
| 长期无纠纷 | +10分 | 近5年无任何纠纷 |
| 征信优秀 | +10分 | 信用评分高，无查询记录 |

**封顶加分**：最多加30分

## 三、风险权重配置

### 风险类别权重

```
司法风险：40%  （最重要）
信用风险：30%
行政风险：20%
其他风险：10%
```

### 计算公式

```
综合风险分 = 100 - (
    司法扣分 × 0.4 +
    信用扣分 × 0.3 +
    行政扣分 × 0.2 +
    其他扣分 × 0.1
)
```

### 时间衰减因子

记录的时间越久，影响越小：

| 时间范围 | 衰减系数 | 说明 |
|----------|----------|------|
| 0-1年 | 1.0 | 完全权重 |
| 1-2年 | 0.8 | 80%权重 |
| 2-3年 | 0.6 | 60%权重 |
| 3-5年 | 0.4 | 40%权重 |
| 5年以上 | 0.2 | 20%权重 |

**应用方式**：
```
实际扣分 = 基础扣分 × 衰减系数
```

## 四、风险分析维度

### 维度1：司法风险深度分析

#### 评估指标
1. **案件数量**：近5年涉案数量
2. **案件类型**：民事/刑事/行政分布
3. **当事人角色**：原告/被告/第三人
4. **案件结果**：胜诉/败诉/和解
5. **执行状态**：已结案/执行中/终本

#### 风险等级判定

| 情况 | 风险等级 |
|------|----------|
| 失信被执行人 | 🔴 高风险 |
| 被执行人且未结案 | 🔴 高风险 |
| 限制高消费 | 🔴 高风险 |
| 刑事案件被告人 | 🔴 高风险 |
| 作为被告败诉3起以上 | 🟠 中风险 |
| 作为被告败诉1-2起 | 🟡 低风险 |
| 仅作为原告胜诉 | 🟢 无风险 |
| 无司法记录 | 🟢 无风险 |

### 维度2：信用风险深度分析

#### 评估指标
1. **信贷账户数量**：贷款、信用卡账户数
2. **逾期记录**：逾期次数、金额、时长
3. **查询记录**：近6个月查询次数
4. **公共记录**：欠税、民事判决等
5. **还款能力**：收入与负债比例

#### 风险等级判定

| 情况 | 风险等级 |
|------|----------|
| 连续逾期90天以上 | 🔴 高风险 |
| 累计逾期6次以上 | 🟠 中风险 |
| 有欠税/税务违法记录 | 🟠 中风险 |
| 近3个月查询超10次 | 🟡 低风险 |
| 1-2次短期逾期 | 🟡 低风险 |
| 无不良记录 | 🟢 无风险 |

### 维度3：行政风险深度分析

#### 评估指标
1. **处罚次数**：行政处罚次数
2. **处罚金额**：累计处罚金额
3. **处罚类型**：警告/罚款/吊销等
4. **企业异常**：经营异常、严重违法
5. **整改情况**：是否已整改

#### 风险等级判定

| 情况 | 风险等级 |
|------|----------|
| 严重违法失信名单 | 🔴 高风险 |
| 吊销许可证 | 🔴 高风险 |
| 重大行政处罚（10万+） | 🟠 中风险 |
| 列入经营异常名录 | 🟠 中风险 |
| 一般行政处罚 | 🟡 低风险 |
| 无行政处罚记录 | 🟢 无风险 |

## 五、风险建议生成规则

### 自动生成建议

根据风险分析结果，自动生成针对性的建议：

#### 🔴 高风险建议

```
【重要提示】该人员存在严重风险记录，建议：

1. 谨慎考虑与此人员的合作/录用
2. 如确需合作，要求提供详细的解释说明
3. 建议增加担保措施或风险控制条款
4. 持续关注相关案件的后续进展
5. 必要时寻求法律专业人士的意见

具体风险点：
- [列出具体风险点1]
- [列出具体风险点2]
...
```

#### 🟠 中风险建议

```
【关注提示】该人员存在一定风险，建议：

1. 进一步了解相关风险的具体情况
2. 要求提供相关案件的结案证明或说明
3. 在合同中增加相应的风险控制条款
4. 定期复查其风险状况
5. 根据具体情况决定是否继续合作

需要关注的问题：
- [列出关注点1]
- [列出关注点2]
...
```

#### 🟡 低风险建议

```
【提示】该人员存在轻微风险记录，建议：

1. 正常关注即可，不影响主要决策
2. 了解风险记录的具体背景
3. 持续观察后续表现
```

#### 🟢 无风险建议

```
【结论】该人员未发现明显风险记录。

建议：可以正常合作或录用。
```

## 六、特殊场景处理

### 场景1：信息不完整

当某些平台无法查询时：
- 在报告中标注"数据缺失"
- 基于已有数据进行评估
- 建议补充其他验证方式

### 场景2：同名同姓

当查询结果可能包含同名他人时：
- 通过身份证号精确匹配
- 在报告中注明可能的同名情况
- 建议人工核实

### 场景3：记录已消除

当负面记录已撤销或过期时：
- 仍记录历史信息
- 标注"已撤销"或"已过期"
- 适当降低风险评分

### 场景4：正在申诉/复议中

当记录处于争议状态时：
- 记录当前状态
- 建议等待最终结果
- 暂缓最终决策

## 八、持续优化

风险评估模型应根据实际使用情况进行持续优化：

1. **收集反馈**：收集用户对评估结果的反馈
2. **调整权重**：根据实际情况调整各项权重
3. **更新规则**：新增风险类型，完善评分规则
4. **校准模型**：与人工评估结果对比，校准模型
